jQuery

jQuery is one of the most common and most powerful libraries that exists for easily programming in the browser.  It is fairly small, simple, and is cross-browser aware so that the programmer using jQuery doesn't have to worry about browser issues.  Technically most everything that jQuery does can be done with plain JavaScript, but it would be much more verbose with a lot more code.   

What is jQuery?

JavaScript is a very flexible and powerful language.  However, it can be a bit unwieldy to write efficient code in, especially when dealing with manipulating data in a web browser. The repeated "getDocumentById" calls and manual manipulation can quickly become overwhelming. There is also the concern (admittedly not as bad as it once was) of cross-browser functionality. Different browsers can sometimes handle certain aspects of the DOM in different ways which can make it tedious to code around.

What is jQuery?

Since jQuery is a library it must be downloaded and included via a <script> tag in the html file.  The latest version can be found on the jQuery website.  There can be several conventions as to where the file should be placed.  In all the examples and lab instructions we will simply assume the jQuery file is in the same location as the web page it is used on.

What is jQuery?

Since a jQuery library file is just a JavaScript file, it can be opened to examine the source code.

  • If the file is not a "min" file it will be readable code, complete with comments and proper formatting.
    • The "min" version is optimized for quick download and is a lot harder to decipher.
    • Functionally, it is equivalent to the full readable version, but in the minified version, all whitespace has been removed, variable and function names shortened, and other optimizations are done to ensure the file is as small as possible and to optimize the download time.

ready() Functions

Inside of this function, other functions can be placed. These functions can refer to DOM elements by using selectors that appear much like CSS selectors. For example the following code will show a simple alert box when the user clicks the link:

<p>This is a basic jQuery page.</p>
<a id="node-link" href="http://nodejs.org">Node</a>
<script
  src="https://code.jquery.com/jquery-3.4.1.min.js"
  integrity="sha256-CSXorXvZcTkaix6Yvo6HppcZGetbYMGWSFlBw8HfCJo="

<script>

ready() Function

Breaking down this code, the $( "#node-link" ) is what selects the element, the .click() is the event to listen for, and the function(event) code defines what happens when the event is triggered - this is known as the "event handler". The parameter event is passed in as an argument from the system and the preventDefault() call stops the execution propagation of the event. In other words, when the link is clicked the alert "Thanks for visiting node!" will be shown, but the user will not be taken to the node page. Instead, if the preventDefault call was left off, the user would be taken to the page after the alert was cleared.

Events

Even in a library meant to be more succinct than JavaScript, the above .click() method is actually a shortcut for a longer event handling mechanism. The "long form" of the event handling function uses the "on" method with the name of the event, like so:

$( "#node-link" ).on("click", function( event ) {
    alert( "Thanks for visiting node!" );
    event.preventDefault();
});

Events

  • Note that in older versions of jQuery the .bind() function was used for essentially the same purpose as .on(). This method is deprecated  and may not be used much any longer, but it is important to know it existed.
  • These shortcut convenience methods exist for many of the common browser events such as .change(), .blur(), .focus(), and .submit(). All of the functions that are specified in the handlers receive the event object as shown earlier. This object has many properties and methods which will be detailed later.

Manipulating the DOM

Adding/Removing Elements

Aside from dynamic visual manipulation, another powerful ability of jQuery is to manipulate the page DOM by adding or removing elements. Using selectors, this can be done to individual elements or groups of elements.

The most basic way is by adding hard-coded content. This involves using a selector and calling the chosen location method with a manually constructed html string. The "location" methods are:

  • .after() - insert content after the selected element.
  • .append() - insert content at the end of the selected element.
  • .before() - insert content before the selected element.
  • .prepend() - insert content at the beginning of the selected element.

Adding/Removing Elements

Note that the selection can be a single element or a collection; if a collection, then the content is added to each element.

AJAX

Webpages were traditionally static markup - the entire page was retrieved on each round trip to the server. Then the technology evolved to the point of having server-side rendering technology to create dynamic content. This is how technologies such as ASP.NET, JSP, Django, Ruby, and several other modern technologies work. However many of these still require a full round trip to the server to refresh the entire page. This can be time consuming and not responsive to the user.

AJAX

  • All the major web browsers and the expansion of JavaScript greatly improved this area through a combination of technologies that eventually became known as AJAX - Asynchronous JavaScript And XML.
  • This technology allowed partial updates of a webpage by using normal http calls combined with JavaScript.
  • Although still part of the name, XML is not the transport of choice. Rather JSON - or JavaScript Object Notation, is used as it is generally lighter and more compact.
  • This section will discuss the AJAX and JSON support built into the jQuery library.

GET and POST

AJAX works by sending requests to the server. There are many "methods" to send data to the server, but probably the two most common are "GET" and "POST". 

  • GET - used for non-destructive retrieveal of data from the server.
  • POST - used to create/update data on the server.

Server applications known as WebAPIs are typically deployed to listen for these requests. These are essentially web sites that are designed specifically to listen for these requests.

  • The code examples will be hitting a WebAPI but the details of constructing that are beyond the scope of this lesson. Just be aware there is a server application listening for web requests and will return data when requested.

Main Method

  • The function in jQuery to initiate an AJAX request is .ajax(). This function has quite a few parameters that are passed in via a JavaScript PlainObject - key/value pairs that configure the request.
  • jQuery also provides a few shortcut convenience methods that eliminates specifying some of those parameters. However the core method will be shown first, then the convenience methods later.

Key Note:

Deferred Interface

The success(), error(), and complete() methods are one way to register callbacks that happen when the request is complete. Another way that is perhaps more popular is to use the result of the .ajax() call which implements something known as the "Promise Interface".

The code in the next slide shows the above request refactored to use this paradigm.

 <div class="title">AJAX Demo</div>
  <div id="list">
 
  </div>
 
  <button id="btnChange">Call API</button><br />
 
  <script src="jquery-3.4.1.min.js"></script>
  <script>
 
  $( document ).ready( function() {
    $("#btnChange").click(function() {
 
      // Using the core $.ajax() method
      $.ajax({
          url: "https://localhost:44392/api/heroes",
          type: "GET",
          dataType : "json"
      })
      .done(function( data ) {
        $.each(data, function(index, value){
          $("#list").append("<div>" + value.name+ "</div>");
        });
      })
 
    })
  })
 
  </script>

Note that the methods are named .done(), fail(), and .always() instead of success, error and complete. The main advantage of these "promise" methods is that they can be chained together more easily than passing an array to the success parameter. For example, the following snippet assigns multiple functions to be fires after the request completes:

$.ajax({
    url: "https://localhost:44392/api/heroes",
    type: "GET",
    dataType : "json"
})
.done(function(data){alert("done #1");})
.done(function(data){alert("done #2");})
.done(function(data){alert("done #3");})
.fail(function(){alert("fail #1");})
