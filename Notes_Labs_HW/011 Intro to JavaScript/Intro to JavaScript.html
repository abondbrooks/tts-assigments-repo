JavaScript (JS) is a programming language that is lightweight and interpreted  by a just-in-time compiler.

What is Javascript?

You may be asking so what does these terms mean? So, Let's take them one at a time.

An interpreted language  is a language for which most of its implementations executes instructions directly and freely, without previously compiling a program into machine code. The opposite of this would be compiled languages which we'll touch on later.

Interpreted Language

A lightweight programming language is defined as one that is designed to have a very small memory footprint, is easy to implement and/or has minimalist syntax and features

Lightweight

JavaScript is an interpreted languages so it uses a Just-In-Time (JIT) compiler to convert source code to machine code at execution time.

Just-In-Time Compiler

Just-In-Time Compilation

JIT compilation is done by the JavaScript engine. It takes the code that you have written and turns it into something that your native browser understands.

Compiled languages are compiled directly into machine code. Unlike interpreters, compiled code executes faster than interpreted languages by eliminating the overhead associated with interpreting code. It's already in a language that the native machine understands.

Compiled Languages

Compile Vs interpreted

Pros & Cons

Compile Vs Interpreted

While many languages are compiled into native machine code yet some are interpreted. Some languages like Java uses combination of these methods. Java is first compiled into what's known as JVM bytecode. This bytecode is then interpreted by the Java Virtual Machine (JVM) at runtime to native machine code. This process allows Java to run on different platforms such as windows, linux and macs.

What are the features of  Javascript?

The Javascript language also supports imperative, object-oriented and declarative (e.g. functional) programming styles.

JavaScript's Programming Styles

Let's look at these programming styles individually.

  • Object-Oriented
  • Imperative
  • Declarative

Object-Oriented Programming

Object oriented programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data, in the form of fields, and code, in the form of procedures or methods. In applications, these objects interact with one another to meet the requirements of the applications being built. In OOP, these objects are often referred to as classes.

Object-Oriented Programming

In object oriented programming, you would abstract the concept of a car into a class or object. Each object having it's own set of features or attributes.

Imperative Programming

In computer science, imperative programming is a programming paradigm in which step by step instructions are given to a computer to solve a specific problem.

The focus of imperative programming is on the how the problem is to be solved. The written code serves as a step-by-step guide to do so.

Declarative Programming

In computer science, declarative programming is a programming paradigm in which the focus is to build the logic of software without describing it's flow.

For example,  in HTML you use <img src="/big-teddybear.jpg> to display an image and you don't care how it does this.

Imperative Vs Declarative

Where to use JavaScript?

In it's early days, JavaScript was mainly used in browsers. Because web pages are static and stateless, JavaScript was used by the browser in order to change the state of simple web pages. The interactions between users and web pages could be enhanced with the use of JavaScript to enable simple things like storing and retrieving data or responding to events that occur in the browser.

Where to use JavaScript?

Today, JavaScript can be found not only in the browser by also on mobile devices,  desktop, IoT devices, in games and on backend servers and more.

Let's explore a few of the places mentioned on the previous slide where JavaScript is used.

  • Web Browser
  • Desktop
  • Web Server

A web browser is actually a computer program. A program that allows you to view web pages and follow hyperlinks from one page to another or one site to another for that matter. Some prominent browsers that you may be familiar with is Google's Chrome, Microsoft's Internet Explorer and Apple's Safari. 

JavaScript in the Web Browser

Web browsers are designed to display content formatted with a language called HyperText Markup Language or HTML for short. It was invented in the 1990s as one of several ways to embed simple logic for data verification and interactivity in web pages. Javascript today is widely used to support this interactivity.

JavaScript in the Web Browser

When using JavaScript with web browsers, normally the JavaScript would be included as a separate file. Browsers have come a long way and now support builtin debugging tools for developers. One such tool is the console which we'll be using shortly.

JavaScript in the Web Browser

Programmers use the console to debug code when debugging JavaScript in web pages.

Otherwise, you save your file and refresh over and over again until you get it right.

Let's try a couple other computational exercises:

3 + 5 * 6
(3 + 5) * 6

In programming, the text we include in the parenthesis is called a string. The quotes enclosing the text is required. We can also do this with the arithmetic expressions that we performed earlier. Enter the following and hit enter to see this in action:

Form Validation

Form Validation

Web pages that include forms often use JavaScript to validate the information entered into the form. Why? Well, when you enter information into a form, it's usually being used elsewhere and we use JavaScript to ensure that the information entered is valid.

For example, we can use JavaScript to ensure that a user enters and confirms passwords in a form.

Form Validation

function emailIsValid (email) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)
}
 
emailIsValid('tyler@tyler@tylermcginnis.com') // false
emailIsValid('tyler@tylermcginnis.com') // true

Responding to Events

element.addEventListener("click", function(){ alert("Hello World!"); });

One of JavaScript most powerful features is the ability to listen for events that may happen on a web page. These events may include clicking on a button, hitting the submit button on a form, detecting when the page has loaded, etc.
